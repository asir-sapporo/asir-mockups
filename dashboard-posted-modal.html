本日投稿の動画 — 「2日分表示」＋「投稿済みモーダル」

・本日投稿リスト: 公開日が 今日＋昨日 の未投稿を表示（投稿し忘れ防止）。昨日分は「期限切れ」。一昨日以前は外れる。
・チェック＝投稿済み（チーム全員で共有・DB保存）。チェックした動画はリストから消え、投稿済みモーダルへ。
・投稿済みモーダル: 今日チェックした分だけ表示。日付が変われば（明日には）モーダルから消える。

動きのまとめ: 公開日が来る → 本日投稿リストに出る（今日＋昨日の2日間）→ 投稿したらチェック → モーダルへ移動（今日のうちだけ表示）→ 明日にはモーダルから消える。チェックを外せば本日投稿リストに戻る（公開日が今日／昨日の範囲内なら）。

確認したいポイント（Q&A）

  1. Q1. 昨日公開分の未投稿は 「06-09 期限切れ」（赤） 表示でOK？（既存の期限切れ表示を流用）
  2. Q2. 開くボタンは「未投稿リストの下」でOK？ 投稿済み0件なら非表示（③）。
  3. Q3. モーダルは 新しい順（投稿時刻の降順）の単純リスト でOK？（今日分だけなので日付の区切りは無し）
  4. Q4. 各行に投稿時刻（14:32）と「未投稿に戻す」ボタンを置く形でOK？
  5. Q5. チェックは 全員共有。＝誰かが投稿済みにしたら全員の画面で投稿済みになる。これでOK？
  6. Q6. 「誰が投稿済みにしたか」は記録だけ持つ（画面非表示）でOK？ 表示も要るなら出せます。

DBテーブル案（新規・本番＆テスト両方に作成）

投稿済みを全員共有・保存するため、プロジェクト1件＝1行の新テーブルを追加。チェック＝行を追加、戻す＝行を削除。モーダルは posted_at が今日の行だけ表示。未投稿リストは「この行が無い＝未投稿」で判定するため、行は2日程度残し古いものは自動掃除（任意）。

create table public.media_posted_marks (
  project_id   text primary key,                 -- 投稿済みの動画（1件1行＝全員共有）
  posted_at    timestamptz not null default now(),  -- 投稿済みにした日時（モーダル表示判定）
  posted_by    uuid references auth.users(id),      -- 操作者（記録用・Q6）
  created_at   timestamptz not null default now()
);
alter table public.media_posted_marks enable row level security;
create policy "read"   on public.media_posted_marks for select using (auth.role() = 'authenticated');
create policy "insert" on public.media_posted_marks for insert with check (auth.role() = 'authenticated');
create policy "delete" on public.media_posted_marks for delete using (auth.role() = 'authenticated');

※ FK・NOT NULL・RLS はテーブル作成時に設定（DB初期設定ルール準拠）。本番DBはユーザーが手動でSQL実行。
